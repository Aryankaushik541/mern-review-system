# 🔑 How to Get Booking.com API Credentials

Complete guide to obtain API access for the Booking.com Guest Review API.

---

## 📋 Prerequisites

Before you can access the Booking.com Guest Review API, you need:

1. ✅ Active property listed on Booking.com
2. ✅ Access to Booking.com Extranet (property management portal)
3. ✅ Property must have received at least a few reviews
4. ✅ Good standing with Booking.com (no violations)

---

## 🚀 Step-by-Step Guide

### **Step 1: Login to Booking.com Extranet**

1. Go to: https://admin.booking.com
2. Login with your property manager credentials
3. Select your property if you manage multiple properties

---

### **Step 2: Find Your Hotel/Property ID**

Your Hotel ID is required for API calls.

**Method 1: From URL**
- Look at the browser URL when logged in
- Example: `https://admin.booking.com/hotel/hoteladmin/properties/123456`
- Your Hotel ID = `123456`

**Method 2: From Property Settings**
- Go to **Settings** → **Property Details**
- Look for **Property ID** or **Hotel ID**
- Copy the number

**Save this ID** - you'll need it for `BOOKING_HOTEL_ID` in `.env`

---

### **Step 3: Request API Access**

Booking.com API access is not automatically available. You need to request it.

**Option A: Contact Booking.com Support**

1. Go to **Help** → **Contact Support** in Extranet
2. Select **Technical Support** or **API Access**
3. Request access to **Guest Review API**
4. Mention:
   - Your property ID
   - Reason: "Need to display guest reviews on our website"
   - Use case: "Integrate reviews into our property management system"

**Option B: Email Booking.com**

Send email to: `connectivity@booking.com`

**Email Template:**
```
Subject: Request for Guest Review API Access - Property ID: [YOUR_PROPERTY_ID]

Dear Booking.com Connectivity Team,

I am writing to request access to the Guest Review API for our property.

Property Details:
- Property Name: [Your Property Name]
- Property ID: [Your Property ID]
- Extranet Email: [Your Email]

Use Case:
We would like to display our Booking.com guest reviews on our property website 
to provide transparency and build trust with potential guests. We plan to use 
the Guest Review API to fetch and display reviews in real-time.

Technical Details:
- We will use the API to fetch reviews only (read-only access)
- Reviews will be displayed on our website with proper Booking.com attribution
- We will implement proper caching to minimize API calls
- We understand and will comply with Booking.com's API usage policies

Please provide:
1. Machine account credentials (username and password)
2. API documentation and usage guidelines
3. Rate limits and best practices

Thank you for your assistance.

Best regards,
[Your Name]
[Your Title]
[Property Name]
[Contact Information]
```

---

### **Step 4: Wait for Approval**

- **Response Time:** Usually 2-5 business days
- **What to Expect:** Booking.com will review your request
- **Approval Criteria:**
  - Property in good standing
  - Valid use case
  - Compliance with terms of service

---

### **Step 5: Receive API Credentials**

Once approved, Booking.com will provide:

1. **Machine Account Username**
   - Format: Usually `property_123456_api` or similar
   - Example: `hotel_987654_machine`

2. **Machine Account Password**
   - A secure password generated by Booking.com
   - Example: `Abc123XyZ789!@#`

3. **API Documentation**
   - Endpoint URLs
   - Authentication method (Basic Auth)
   - Rate limits
   - Usage guidelines

---

### **Step 6: Configure Your Application**

Edit `backend/.env`:

```env
# Booking.com Configuration
BOOKING_HOTEL_ID=123456
BOOKING_API_USERNAME=property_123456_api
BOOKING_API_PASSWORD=your_secure_password_here
```

**⚠️ IMPORTANT:**
- Never commit `.env` to version control
- Keep credentials secure
- Don't share credentials publicly
- Rotate passwords periodically

---

## 🔒 Security Best Practices

### **1. Secure Storage**
```env
# ✅ Good - In .env file (not committed)
BOOKING_API_USERNAME=property_123456_api
BOOKING_API_PASSWORD=SecurePassword123!

# ❌ Bad - Hardcoded in code
const username = 'property_123456_api';
const password = 'SecurePassword123!';
```

### **2. Environment Variables**
```javascript
// ✅ Good - Use environment variables
const username = process.env.BOOKING_API_USERNAME;
const password = process.env.BOOKING_API_PASSWORD;

// ❌ Bad - Hardcoded

```


## 📊 API Endpoint Details

### **Base URL:**
```
https://supply-xml.booking.com/review-api/properties
```

### **Get Reviews Endpoint:**
```
GET https://supply-xml.booking.com/review-api/properties/{hotel_id}/reviews
```

### **Authentication:**
```
Basic Auth
Username: [Your machine account username]
Password: [Your machine account password]
```

### **Headers:**
```
Accept: application/json
Content-Type: application/json
```

### **Query Parameters:**
- `limit` - Number of reviews (default: 10, max: 100)
- `next_page` - Pagination cursor for next page

## 🐛 Troubleshooting

### **Problem 1: 401 Unauthorized**

**Possible Causes:**
- Incorrect username or password
- Credentials not activated yet
- Machine account expired

**Solutions:**
1. Double-check credentials in `.env`
2. Verify no extra spaces in credentials
3. Contact Booking.com support to verify account status
4. Request new credentials if needed

---

### **Problem 2: 403 Forbidden**

**Possible Causes:**
- API access not granted
- Property ID mismatch
- Account permissions insufficient

**Solutions:**
1. Verify you have API access approval
2. Check hotel ID is correct
3. Contact Booking.com to verify permissions

---

### **Problem 3: 404 Not Found**

**Possible Causes:**
- Incorrect hotel ID
- Wrong API endpoint URL
- Property not found

**Solutions:**
1. Verify hotel ID from Extranet
2. Check API endpoint URL is correct
3. Ensure property is active on Booking.com

---

### **Problem 4: No Reviews Returned**

**Possible Causes:**
- Property has no reviews yet
- Reviews not published
- API returning empty array

**Solutions:**
1. Check if property has reviews in Extranet
2. Wait for reviews to be published (can take 24-48 hours)
3. Try with different limit parameter

## 📝 Important Notes

1. **API Access is NOT Automatic**
   - You must request and be approved
   - Not all properties may be eligible

2. **Machine Account vs Regular Account**
   - Machine accounts are for API access only
   - Don't use your regular Extranet login credentials

3. **Rate Limits**
   - Booking.com may impose rate limits
   - Implement caching to reduce API calls
   - Typical limit: 100-1000 requests per hour (varies)

4. **Review Moderation**
   - Reviews go through Booking.com moderation
   - May take 24-48 hours to appear
   - Some reviews may be rejected

5. **Terms of Service**
   - Must display Booking.com attribution
   - Cannot modify review content
   - Must comply with Booking.com policies
